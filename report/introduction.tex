In August 2025, Anthropic's Claude model was reportedly used in one of the first documented cases of ``vibe hacking'' as outlined in \fullcite{ThreatReport2025}. In that case, the attacker used the model to collect publicly available vulnerability information, identify organizations using the affected software, and exploit the vulnerability to gain access to internal infrastructure. The incident was notable because it required limited human involvement, while the model assisted in producing malicious tooling used for extortion.

If AI continues to develop at the current pace, similar incidents are likely to become more frequent. It is also plausible that some future attacks will be increasingly autonomous, if not fully. In that context, the need for stronger cybersecurity capabilities will grow, and many experts argue that AI will play a central role in both attack and defense strategies. This motivates research into how AI models behave when they share the same system and are tasked with detecting, disrupting, or countering each other.

\subsection{The Problem}
\label{ResearchQuestion}

The aim of this research paper is not to predict exactly how AI models would counteract each other, but to explore how they might use available system tools in a simulated local environment, and whether useful insights can be gained from this process.

In simple terms, the problem is twofold. First, it is unclear which terminal commands are realistically available to an AI agent running locally with broad system access. Second, it is not known how an AI model should be trained to use such commands effectively, or which combinations are most useful for detecting and counteracting another model in the same environment.

\subsubsection{Research Question}

The research question is then as follows:

\begin{center}
    \textit{How does AI interact with other AI models in a local system, and what are the implications for detecting and counteracting malicious AI?}
\end{center}

\begin{itemize}
    \item \textbf{RQ1:} What commands are available for an AI model to use in a local system, and how can they be used to interact with other AI models?
    \item \textbf{RQ2:} How can an AI model be trained to use these commands effectively to detect and counteract another AI model that exists in the same local system?
    \item \textbf{RQ3:} What does a simulation of two AI models interacting with each other in a local system look like, and what can we learn from it?
    \item \textbf{RQ4:} What are the implications of AI models interacting with each other in a local system for the future of AI safety and security?
    \item \textbf{RQ5:} What are the ethical considerations of AI models being trained on data generated by other AI models?
\end{itemize}

To address these questions, we will first compile a list of terminal commands on linux that can be used to outline a base of actions for the simulation. This will be limited to linux for the sake of simplicity, but the process can be expanded to include Windows commands as well.

\subsection{Linux Command Inventory}

A list of the linux commands used in this paper is included in \autoref{AppendixLinuxCommands}. This list is not an exhaustive list of all commands available on linux based distributions, it is based on the commands found on \fullcite{CommandLineReference}, which includes all the built-in commands for the bash shell. For a full list of linux commands, see \fullcite{LinuxManualPages}.

For the sake of the keeping the simulation simple, we will focus on a subset of commands that most obviously relate to the problem at hand. Some of the commands included in the list are not directly related to the problem, however they pose interesting possibilities for future research that will be brought up in the discussion section of this paper.

To begin with, we can subdivide the commands into the underlying categories of actions:

\begin{itemize}
    \item \textbf{Navigation and Search}
    \item \textbf{File Management}
    \item \textbf{Scheduling}
    \item \textbf{Coding and Scripting}
    \item \textbf{Partitioning and Filesystem Management}
    \item \textbf{Process Management}
    \item \textbf{Network Management}
    \item \textbf{User Management}
\end{itemize}

It is important to note that these categories are neither definitive nor exhaustive. There are a great many commands that we will not be able to cover in this paper, in addition to several commands being able to fit into multiple categories. The categorization is meant for ease of reference and to provide a framework for understanding the different types of actions that could potentially be taken by an AI model in a local system.

\subsubsection{Navigation and Search}

This category includes commands that allow for navigation of filesystems and searching for files or content within them. The following commands are included for this reason:

\begin{itemize}
    \item \texttt{cd} --- Changes the current working directory. Included because an agent operating in a local system must be able to move between directories to access different parts of the filesystem.
    \item \texttt{pwd} --- Prints the current working directory. Included because it allows an agent to determine its current location in the filesystem at any point.
    \item \texttt{ls} --- Lists the contents of a directory, including files and subdirectories. Included because it allows an agent to enumerate the contents of any directory it has access to.
    \item \texttt{dir} --- Provides a brief listing of directory contents. Included as a functional alternative to \texttt{ls} for directory enumeration.
    \item \texttt{pushd}, \texttt{popd}, \texttt{dirs} --- Manage a stack of remembered directories, allowing an agent to save its current location, navigate elsewhere, and return. Included because they enable structured traversal of the filesystem.
    \item \texttt{find} --- Searches for files and directories meeting specified criteria such as name, type, permissions, or modification time. Included because it allows an agent to locate files of interest across the entire filesystem.
    \item \texttt{grep}, \texttt{egrep}, \texttt{fgrep} --- Search the contents of files for lines matching a given pattern, extended regular expression, or fixed string, respectively. Included because they allow an agent to scan files for specific content.
    \item \texttt{locate} --- Finds files by name using a pre-built index of the filesystem. Included because it provides a fast alternative to \texttt{find} for locating files by name.
    \item \texttt{awk}, \texttt{gawk} --- Scan and process text files, with support for pattern matching and field-based operations. Included because they allow an agent to extract and filter structured information from files.
    \item \texttt{look} --- Displays lines from a file that begin with a given string. Included because it provides a simple means of searching sorted text data.
    \item \texttt{basename}, \texttt{dirname} --- Extract the filename and directory components of a path, respectively. Included because they allow an agent to manipulate and interpret file paths.
    \item \texttt{less}, \texttt{more} --- Display file contents one screen at a time. Included because they allow an agent to read through large files in a controlled manner.
    \item \texttt{head}, \texttt{tail} --- Output the first or last lines of a file, respectively. Included because they allow an agent to quickly inspect the beginning or end of a file without reading it in full.
\end{itemize}
